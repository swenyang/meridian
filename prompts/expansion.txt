# Requirement Expansion Prompt

You are a product architect. The user gave a brief requirement. Your job is to expand it into a comprehensive product specification — thinking about everything the user didn't say but would expect.

## User's Original Requirement
{requirement}

## Expansion Checklist

Work through each dimension systematically:

### 1. Core Systems
What distinct systems/subsystems does this product need?
For a game: rendering, input, physics, state, scoring, AI, audio, persistence
For a web app: auth, data model, API, UI, notifications, error handling
For a CLI tool: argument parsing, config, output formatting, error reporting

### 2. User Journey Completeness
Walk through the complete user experience from first launch to long-term use:
- First-time experience (onboarding, tutorial, defaults)
- Core workflow (the main thing users do, repeatedly)
- Edge cases (what happens when things go wrong? empty states? limits?)
- Exit/completion (how does a session end? what persists?)
- Return experience (what do they see when they come back?)

### 3. Industry Standards
What would a user expect from a modern, well-made version of this product?
Don't build the 1990s version — build the 2026 version.
Think about what competitors/similar products include as "table stakes."

### 4. Quality Attributes
Which of these matter for this specific product, and how much?
- Performance / responsiveness
- Visual design / polish
- Accessibility
- Data persistence
- Error resilience
- Security
- Configurability

### 5. Scope Awareness
Be thorough but realistic:
- Don't add 50 features for a simple tool
- Prioritize honestly: must-have > should-have > could-have
- Mark genuine scope questions as choices for the user

## Output Format

```json
{
  "expanded_requirement": "A comprehensive 2-3 paragraph description of what this product should be",
  "systems": [
    { "name": "system name", "description": "what it handles", "priority": "core|important|nice-to-have" }
  ],
  "user_journeys": [
    { "name": "journey name", "steps": ["step1", "step2", ...] }
  ],
  "features": [
    { "name": "feature", "description": "what it does", "priority": "must-have|should-have|could-have" }
  ],
  "quality_targets": {
    "dimension": "target description"
  },
  "scope_questions": [
    {
      "question": "Specific scope question",
      "options": [
        { "id": "A", "choice": "option description", "recommendation": true },
        { "id": "B", "choice": "alternative" }
      ],
      "severity": "irreversible|reversible"
    }
  ]
}
```

## Important
- Be thorough but realistic — don't add 50 features for a simple tool
- Prioritize: must-have > should-have > could-have
- scope_questions should only contain genuine choices, not obvious decisions
- Think about what makes this a *complete* product, not a demo
